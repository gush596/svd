# SVD 量化项目分析报告

## 1. 算法修正

### 原始理解（错误）
- 最终残差以 4-bit 量化
- 每轮 rank/u/s/v 可以不同
- 公式: `eff = Σ(...) / total + 4`

### 正确算法
- **最终残差舍弃**，不量化
- **每轮 rank、u_bits、s_bits、v_bits 固定不变**
- 公式: `eff = n_rounds × rank × (out × u_bits + s_bits + in × v_bits) / (out × in)`

## 3. MSE 表现分析

### 关键发现

1. **量化精度至关重要**: u3s4v3 效果远好于 u2s2v2
2. **更高 rank 不一定更好**: rank=1 (170轮) 效果接近 rank=8 (21轮)
3. **残差舍弃模式在低秩数据上效果好**: 比直接 4-bit 好 44%~53%

### 效果对比（256×256, 4.0 bit 预算）

| 方法 | MSE | 比直接4-bit |
|------|-----|------------|
| 直接 4-bit | 0.04557 | baseline |
| 迭代SVD rank=1 u3s4v3 | 0.02562 | **好 44%** |
| 迭代SVD rank=4 u3s4v3 | 0.02399 | **好 47%** |
| 迭代SVD rank=8 u3s4v3 | 0.02155 | **好 53%** |

### 量化精度影响

| 配置 | MSE ratio | 说明 |
|------|-----------|------|
| u3s4v3 | 0.562x | ✅ 足够精度，效果好 |
| u2s3v2 | 0.923x | ⚠️ 勉强 |
| u2s2v2 | 1.741x | ❌ 精度不足，效果差 |

## 4. 代码修改

### 核心变更

```python
# 旧算法（已废弃）
# 最终残差 4-bit 量化，每轮配置可不同
residual_q = quantize_mse(residual, n_bits=4, group_size=group_size)
W_approx = W_approx + residual_q
eff = (svd_bits_total + total_params * 4) / total_params

# 新算法（正确）
# 最终残差舍弃，每轮配置固定
# W_approx 就是最终结果，不加残差
eff = svd_bits_total / total_params
```

### API 变更

```python
# 旧 API
iterative_residual_svd(W, group_size=128, max_eff_bits=4.0, rounds=[...])

# 新 API
iterative_residual_svd(W, group_size=128, max_eff_bits=4.0,
                       rank=1, u_bits=3, s_bits=4, v_bits=3,
                       n_rounds=None)  # None=自动计算
```

## 5. 固有特性

### 优势
- 等效 bit 可以严格 ≤ 4.0（不含残差存储）
- 在低秩数据上效果显著优于直接量化
- 灵活调整 rank 和量化精度的权衡

### 限制
- 对非低秩数据效果差（残差舍弃丢失信息）
- 每轮 SVD 计算开销大（rank=1, 768×768 需要 511 轮）
- 量化精度过低（u2s2v2）会导致严重退化

## 6. 实验报告数据对比

原报告中的数据基于旧算法（残差 4-bit 量化），新算法（残差舍弃）表现不同：

| 项目 | 旧算法 | 新算法 |
|------|--------|--------|
| 残差处理 | 4-bit 量化 | 舍弃 |
| 等效 bit | > 4.0 (含残差) | ≤ 4.0 (不含残差) |
| MSE 表现 | 改善 50% | 改善 44%~53% |
| 适用约束 | ≤4-bit 不满足 | ≤4-bit 满足 ✅ |
