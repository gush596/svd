# SVD 量化实验报告

## 实验环境
- 模型: OPT-125M (125M 参数, 12层 Transformer)
- 数据集: wikitext-2, 5 samples, seqlen=512
- FP Baseline PPL: 37.34
- 量化约束: 等效 ≤4-bit, group_size ≥128
- CPU only, PyTorch 2.12.1, transformers 5.12.1

## 方法 2: 重要值保护

识别绝对值最大的 top-k% 权重，用高精度量化，其余用标准精度。

**⚠️ 2026-06-26 修正：原公式忽略了位置 bitmap 存储开销**

~~旧公式: `eff = (1 - ratio) × n_bits + ratio × protection_bits`~~

新公式（含 bitmap + scale 开销）:
```
eff_values = (1 - ratio) × n_bits + ratio × protection_bits
eff_bitmap = 1.0                           # 每参数 1 bit 标记是否为重要值
eff_scale  = (n_groups_imp + n_groups_rem) × 32 / n  # 两组量化各需 group 级 scale
eff_total  = eff_values + eff_bitmap + eff_scale
```

bitmap 是必须的：否则解码时无法区分哪些参数用了高精度。bitmap 1 bit/param 是下限（索引方案在 ratio<50% 时更贵）。

**结论：重要值保护在 ≤4-bit 约束下不可行。** 最低 eff 也在 5.3+ 以上。

## 方法 3: SVD Hybrid

SVD 低秩重建 + 残差量化。

```
W_approx = quantize(U_k, svd_bits) @ quantize(V_k, svd_bits) + quantize(residual, 4)
eff = rank × (out+in) × svd_bits / (out×in) + residual_bits
```

**注意**: 此方法的 eff > 4.0（含残差 4-bit 存储），无法满足 ≤4-bit 约束。

## 方法 4: 迭代残差 SVD (残差舍弃模式)

**核心算法**:
1. 对当前残差做 SVD，保留 top-rank 个奇异值
2. 分别量化 U, S, V（每轮 rank、u_bits、s_bits、v_bits 固定不变）
3. 残差 = 上一轮残差 - 本轮分量
4. 重复步骤 1-3，直到达到等效 bit 预算
5. **最终残差舍弃，不量化**

```
for round in range(n_rounds):
    U, S, Vt = svd(residual)
    W_approx += quantize(U[:,:rank], u_bits) @ diag(quantize(S[:rank], s_bits)) @ quantize(Vt[:rank,:], v_bits)
    residual -= component
# 最终残差舍弃
```

### 等效 bit 公式

```
eff = n_rounds × rank × (out × u_bits + s_bits + in × v_bits) / (out × in)
```

**不含残差项**，可以严格 ≤ 4.0。

### 每轮 bit 开销

```
每轮 bits = rank × (out × u_bits + s_bits + in × v_bits)
每轮 eff  = 每轮 bits / (out × in)
```

## 方法 5: 迭代残差 SVD - u4v4 / s不量化 (2026-06-25)

### 算法改进

1. **s 不量化**: 奇异值 S 用 float32 存储（32 bit/值），不做量化，避免精度损失
2. **综合等效 bit**: 在原始等效 bit 基础上，加入 scale/zero-point 存储开销
3. **GPU 自动检测**: 有 GPU 时自动使用 GPU 加速 SVD 分解

### 综合等效 bit 公式

```
原始 eff (eff_raw):
  eff_raw = n_rounds × [rank × (out × u_bits + in × v_bits) + rank × s_bits_eff] / (out × in)
  其中 s_bits_eff = s_bits (量化时) 或 32 (float32 不量化时)

综合 eff (eff_full, 含 scale 存储):
  每轮 U 的 group 数: ceil(out × rank / group_size)
  每轮 V 的 group 数: ceil(rank × in / group_size)
  每轮 S 的 group 数: 1 (如果量化) 或 0 (不量化)
  每轮 scale bits = (n_groups_u + n_groups_s + n_groups_v) × 32
  eff_full = (svd_bits_total + scale_bits_total) / (out × in)
```

**关键发现**: scale 存储开销巨大！group_size=128 时，eff_full 约为 eff_raw 的 2 倍。

### 实验 1: s 量化 vs s 不量化 (rank=4, u4v4, 768×768)

| u4 s3 v4 | 47 | 1.959 | 3.920 | 0.111475 | 2.332x |
| u4 s2 v4 | 47 | 1.959 | 3.920 | 0.145575 | 3.045x |
| **u4 s-fp32 v4 (s不量化)** | **47** | **1.969** | **3.927** | **0.095503** | **1.997x** |

✅ **s 不量化效果最好**: 虽然 s 用 float32 存储（32 bit/值），但因为 rank 通常很小（4-32），
额外开销微乎其微，却显著提高了精度。

### 关键结论

1. **s 不量化是最优策略**: 奇异值数量少（=rank），float32 存储开销可忽略，但避免了量化误差传播
2. **高 rank + 少轮数 > 低 rank + 多轮数**: rank=32 只需 9 轮就接近直接量化水平
3. **scale 存储是主要开销**: eff_full ≈ 2 × eff_raw，说明每个量化参数需要约 1 bit 的 scale 开销
4. **u3v3 性价比高**: 在 eff_full 约束下，u3v3 可以跑更多轮，效果接近 u4v4
5. **计算效率**: 高 rank 更快（rank=32 仅 3.3s，rank=4 需 16.3s）

### 🔴 关键发现：残差舍弃模式在真实模型上严重失败

**问题分析**：
1. 合成数据 MSE 测试效果好（0.989x），但 PPL 测试灾难性失败（837 vs 43）
2. 原因：合成数据有强低秩结构，真实模型权重更复杂
3. 残差舍弃丢失了大量对 PPL 至关重要的高频信息
4. 9 轮 × rank=32 只覆盖了权重的一小部分能量
5. rank=16 更差（PPL=2872）：更低 rank 每轮捕获更少能量，总覆盖不足

**结论：残差舍弃模式在真实模型上不可行。必须量化残差或采用其他策略。**

**方向**：
- 残差不能简单舍弃，需要量化存储
- 或者大幅增加轮数（但 eff 预算不允许）
- 需要结合其他方法（重要值保护 + SVD 混合）

## 方法 7: 异常值 SVD + 残差量化 (2026-06-26)

### 算法

```
1. 从 W 中筛选 top-k% 异常值（按绝对值）
2. 构造稀疏异常值矩阵 W_outlier（异常值保留原位，其余为 0）
3. 对 W_outlier 做迭代残差 SVD（u=3, v=3, s-fp16）
4. 残差 = W - W_outlier_approx
5. 对残差做 n-bit 直接量化

W_approx = W_outlier_svd + W_residual_quantized
```

### 等效 bit 公式

```
SVD 部分:
  eff_raw(SVD) = n_rounds × rank × (out × u_bits + in × v_bits + s_bits_eff) / (out × in)
  eff_full(SVD) = (svd_bits_total + scale_bits_total) / (out × in)

残差部分:
  quantize_mse 对全矩阵做 group 量化，每 group 一个 float32 scale
  residual_eff_full = (total_params × residual_bits + n_groups × 32) / total_params

总等效 bit (残差在 ALL 位置存储，不能省略):
  total_eff_raw  = eff_raw(SVD)  + residual_bits
  total_eff_full = eff_full(SVD) + residual_eff_full
```

**⚠️ 不能省略 outlier 位置的残差！**
residual = W - W_outlier_approx，在 outlier 位置 residual = 原始异常值 - SVD近似 ≠ 0。
残差必须在所有位置存储。

### 实验 1: 不同 outlier_ratio × svd_eff × residual_bits (768×768)

合成数据: rank-96 低秩 + 1% 大幅异常值

| outlier_ratio | svd_eff | res_bits | 轮数 | eff_full | MSE | vs Direct |
|--------------|---------|----------|------|----------|-----|----------|
| 0.05 | 1.0 | 3 | 13 | 4.20 | 0.097671 | 2.0428x |
| 0.10 | 1.0 | 3 | 13 | 4.20 | 0.089823 | 1.8786x |
| 0.15 | 1.0 | 3 | 13 | 4.20 | 0.086193 | 1.8027x |
| 0.20 | 1.0 | 3 | 13 | 4.20 | 0.082639 | 1.7284x |
| 0.30 | 1.0 | 3 | 13 | 4.20 | 0.072914 | 1.5250x |

**❌ residual_bits=3 全部不敌直接 4-bit**

### 实验 2: residual_bits=4 结果 (修正后)

| outlier_ratio | svd_eff | res_bits | 轮数 | true_eff_full | MSE | vs Direct |
|--------------|---------|----------|------|--------------|-----|----------|
| 0.10 | 0.30 | 4 | 4 | 4.54 | 0.028193 | **0.8803x** ✅ |
| 0.10 | 0.50 | 4 | 6 | 4.69 | 0.026456 | **0.8261x** ✅ |
| 0.15 | 0.50 | 4 | 6 | 4.69 | 0.026441 | **0.8256x** ✅ |
| 0.25 | 0.50 | 4 | 6 | 4.69 | 0.026525 | **0.8282x** ✅ |

**⚠️ true_eff_full 全部 > 4.0，不满足 ≤4-bit 约束！**

### 🔥 修正后的结论

**残差必须在全矩阵存储（outlier 位置的残差 ≠ 0），因此:**

1. **residual_bits=3 + SVD**: true_eff 可以 ≤ 4.0，但 MSE 是直接 4-bit 的 2 倍，不可用
2. **residual_bits=4 + SVD**: MSE 好（比直接 4-bit 低 17%），但 true_eff > 4.0，不满足约束
3. **根本矛盾**: 残差覆盖整个矩阵，4-bit 残差本身已 ≈ 4.25 bit/param（含 scale），加上 SVD 必然超 4.0

**等效于：4-bit 残差 ≈ 直接 4-bit 量化 + 额外 SVD 存储开销。SVD 只有在能替代部分残差精度时才有意义。**

### 下一步方向

- 探索更低 residual_bits（2-3 bit）+ 更高 SVD 覆盖的组合
- 或者改变策略：不量化残差，而是用 SVD 覆盖更多能量（高 rank/多轮），同时降低每轮 bit
- 或者放弃“残差量化”思路，回归方法 5/6 的迭代残差 SVD（残差舍弃/不量化）

## 方法 8: 迭代异常值 SVD + 3-bit 残差量化 (2026-06-26)

### 算法

```
for each round:
    1. 从当前残差 R 中取 top-k% 异常值（按绝对值）
    2. 构造稀疏异常值矩阵，SVD 分解，取 rank
    3. 量化 U/V（S fp16），得到本轮分量 W_i
    4. 更新残差 R = R - W_i
    重复直到 SVD eff_raw 达到上限
最终残差做 3-bit 直接量化

W_approx = sum(W_outlier_svd_i) + W_residual_3bit
```

### 等效 bit

```
不计 scale: total_eff_raw = svd_eff_raw + 3.0
目标: total_eff_raw ≤ 4.0 → svd_eff_raw ≤ 1.0
```

### NumPy MSE 测试 (768×768)

| ratio | svd_eff | rank | u/v | 轮数 | eff_raw | MSE | vs 4-bit | vs 3-bit |
|-------|---------|------|-----|------|---------|-----|----------|----------|
| 0.10 | 0.50 | 4 | 3/3 | 15 | 3.48 | 0.0609 | 1.90x | **0.50x** |
| 0.10 | 1.00 | 4 | 3/3 | 31 | 3.98 | 0.0411 | 1.28x | **0.50x** |
| 0.15 | 1.00 | 4 | 3/3 | 31 | 3.98 | 0.0403 | 1.26x | **0.49x** |

比直接 3-bit 好一倍，但比直接 4-bit 差 25~28%。

### PPL 测试 (OPT-125M, CPU)

| 配置 | PPL | Delta | eff_raw | 耗时 |
|------|-----|-------|---------|------|
| FP baseline | 37.34 | - | - | - |
| Direct 4-bit | 43.32 | +5.98 | 4.0 | - |
| Direct 3-bit | 167.11 | +129.77 | 3.0 | - |
| ratio=0.10 svd_eff=0.50 rank=4 u3v3 | 56.23 | +18.89 | 3.48 | 969s |
| ratio=0.10 svd_eff=1.00 rank=4 u3v3 | 49.21 | +11.87 | 3.98 | 1811s |

### 分析

1. **比直接 3-bit 好很多**: PPL 56.23 vs 167.11（好 3 倍），说明迭代异常值 SVD 有效提取了关键信息
2. **比直接 4-bit 差**: PPL 49.21 vs 43.32（差 13.6%），SVD 部分覆盖的能量还不够
3. **eff_raw=3.98 接近 4.0 上限**: 31 轮 × rank=4 的 SVD 约贡献 1 bit，加上 3-bit 残差 = 3.98
4. **SVD 不收敛问题**: outlier_ratio=0.15 时部分层 SVD 不收敛，需添加保护

### 下一步

- 增加 SVD 轮数或 rank 提高覆盖率（但 eff 预算有限）
- 探索更高 u/v bit（u4v4）+ 更少轮数的组合
- 或者在 outlier 位置用更高精度量化而非 SVD
